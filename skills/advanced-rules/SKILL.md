# Skill 12 — 고급 규칙 (§26 7단계)

참조: GAME_SPEC.md §12(후속 규칙·배신), §16(공개 계약 보너스), §25(TODO 2·3·4·6), §29(MVP 유예 목록)

> **착수 조건: 3~5단계(M3~M5) 완료 + 4인 플레이테스트 1회 이상.**
> 이 스킬의 항목이 다른 스킬 작업 중에 필요해 보여도 미리 구현하지 않는다 — no-op 훅만 남긴다.

## 구현 항목 (우선순위 순)

### 1. 단일화 공개 조건 계약
- 제안 terms에 조건 첨부: 자금 이전, VP 약속, **정책 양보**(policy.ts 3순서 훅 연결)
- 공개 계약은 정산 시 엔진이 자동 강제 이행. 이행 수는 최종 보너스(§16-5: 단독 1등 +2)로 연결 — finalScoring의 0 고정값 교체

### 2. 비밀 pact + 배신
- 당사자만 아는 pact 체결 (마스킹: projectView가 당사자 외 제거)
- 정산 시 배신 선택 가능. 배신 시(§12): 배신자 reputation -2, 피해자 유권자 이벤트 2장, `betrayed-secret-pact` 플래그
- 협상 UI: 개인 화면에 pact 제안/수신 패널 (정교한 협상 채팅은 그 다음)

### 3. 이벤트 카드 효과 (30+30장)
- card-data의 판별 유니온 효과 타입에 대한 reducer 스위치 구현
- `useEvent` 액션 활성화, 손패 상한·사용 타이밍 규칙 확정 필요 *(스펙 미정 — 구현 전 결정하고 GAME_SPEC.md 부록에 추가)*

### 4. 후보 능력/약점 확장
- `active: false`인 능력들을 순차 활성화. 능력 1종 = 테스트 1개 세트
- `poll` 액션 활성화 (비용·효과 확정 후 — 부록 A-4 잠정안에서 시작)

### 5. 밸런스 시뮬레이션
- 랜덤/휴리스틱 bot으로 4인 5라운드 N회 자동 대전 스크립트 (`packages/engine/sim/`)
- 리포트: 승자 점수 분포(목표 30-45), 점수 소스 비중, 의제별 달성률, 킹메이커 발동률
- constants.ts 수치 조정 ↔ 시뮬레이션 반복의 근거 자료

## 수용 기준
- 공개 계약 불이행 코드 경로가 존재하지 않음 (자동 집행)
- pact 데이터가 당사자 외 뷰 payload에 없음
- 배신 페널티 3종이 §12 명세와 일치
- 시뮬레이션 리포트가 CI에서 재현 가능 (seed 고정)
